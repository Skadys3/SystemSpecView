# System Specs Viewer (Просмотр характеристик компьютера)

Лёгкое настольное приложение для Windows, предназначенное для просмотра
характеристик компьютера и конфигурации системы: операционная система,
процессор, память, диски, сетевые адаптеры, видеокарта,
материнская плата/BIOS и батарея. Также включает встроенный
**диспетчер задач** — просмотр запущенных процессов с их загрузкой
ЦП/ОЗУ/диска/сети, завершение процессов и окно подробных свойств.

**Примечание:** графический интерфейс приложения, а также все
комментарии/докстринги в исходных файлах — на русском языке.
Идентификаторы кода (имена функций/классов/переменных) остаются на
английском — это стандартная практика: переведён только текст,
видимый пользователю, и комментарии. Этот README оставлен на английском
для справки.

## Возможности

- **Интерфейс с вкладками** (Обзор, ОС, Процессор, Память, Диски, Сеть,
  GPU, Материнская плата/BIOS, Батарея, Диспетчер задач)
- **Диспетчер задач**:
  - таблица всех запущенных процессов с сортировкой по любому столбцу
    (клик по заголовку) и поиском по имени/PID;
  - загрузка ЦП (нормализована к 0–100% по всей системе, как в
    современном Диспетчере задач Windows), память (МБ и %), скорость
    чтения/записи на диск (КБ/с) и число активных сетевых подключений —
    всё обновляется раз в секунду;
  - **завершение процесса** (корректное или принудительное) через кнопку,
    двойной клик или контекстное меню правой кнопкой мыши, с
    подтверждением перед действием;
  - **окно свойств процесса** — командная строка, путь к исполняемому
    файлу, рабочий каталог, родительский процесс, приоритет, время ЦП,
    память RSS/VMS, суммарный объём чтения/записи на диск и др.
- **Живое обновление** загрузки процессора и памяти каждые 2 секунды
  (можно включать/выключать)
- **Неблокирующий интерфейс** — сбор данных выполняется в фоновом потоке,
  поэтому окно не зависает во время получения информации
- **Экспорт отчёта** в `.json` (структурированный, включает полный список
  процессов) или `.txt` (в удобочитаемом виде, включает топ-10 процессов
  по загрузке ЦП)
- **Плавная деградация** — если значение не удаётся прочитать (например,
  из-за прав доступа, отсутствующего датчика или не-Windows ОС), поле
  показывает `N/A` вместо падения приложения
- **Логирование** в `logs/system_specs_viewer.log` для диагностики

## Структура проекта

```
system_specs_viewer/
├── main.py          # Точка входа: настройка логирования, DPI-осведомлённость, запуск GUI
├── models.py         # Датаклассы, описывающие каждую категорию системной информации
├── collectors.py      # Слой сбора данных (psutil + wmi)
├── gui_app.py         # Tkinter GUI (вкладки, потоки, экспорт)
├── requirements.txt
└── README.md
```

Слой сбора данных (`collectors.py`) и слой графического интерфейса
(`gui_app.py`) полностью разделены: GUI никогда не вызывает `psutil` или
`wmi` напрямую, он взаимодействует только с `SystemInfoCollector`. Это
упрощает тестирование, расширение кода или перенос на другой GUI-тулкит
в будущем, если понадобится.

## Источники данных

| Категория                | Источник                          |
|---------------------------|------------------------------------|
| ОС, время работы           | `platform`, `psutil.boot_time`     |
| Загрузка/частота процессора | `psutil`                          |
| Название процессора (подробное) | WMI `Win32_Processor` (резерв: `platform.processor()`) |
| Память / файл подкачки       | `psutil`                          |
| Диски                       | `psutil`                          |
| Сеть                        | `psutil`                          |
| GPU                         | WMI `Win32_VideoController`        |
| Материнская плата / BIOS     | WMI `Win32_BaseBoard`, `Win32_BIOS` |
| Батарея                     | `psutil.sensors_battery()`         |
| Процессы (диспетчер задач)   | `psutil.process_iter`/`Process.*`, `psutil.net_connections` |

Видеопамять GPU: поле `Win32_VideoController.AdapterRAM` в WMI является
знаковым 32-битным значением, поэтому оно переполняется на картах с
4 ГБ видеопамяти и более. Приложение обнаруживает это и показывает
`0.0 GB` вместо бессмысленного отрицательного числа — это известное
ограничение самого WMI, а не то, что можно исправить на стороне клиента.

### Ограничения диспетчера задач

- **Сеть** — psutil не предоставляет кроссплатформенного способа узнать
  скорость сетевого трафика (байт/с) в разрезе процесса ни на одной ОС.
  Столбец «Сеть» показывает число активных сетевых подключений процесса —
  это полезное приближение к сетевой активности, но не пропускная
  способность.
- **Завершение системных/защищённых процессов** может потребовать запуска
  приложения от имени администратора — в этом случае появится
  соответствующее сообщение об ошибке вместо падения приложения.
- **Загрузка ЦП на процесс** нормализована к шкале 0–100% для системы в
  целом (делением на число логических ядер), а не к шкале "0–100% на одно
  ядро", которую по умолчанию возвращает psutil — это сделано для
  соответствия привычному поведению Диспетчера задач Windows.

## Сборка отдельного .exe (опционально)

Если вы хотите распространять приложение без необходимости установки
Python пользователями, используйте
[PyInstaller](https://pyinstaller.org/):

```powershell
pip install pyinstaller
pyinstaller --onefile --windowed --name "SystemSpecsViewer" main.py
```

Готовый исполняемый файл появится в `dist\SystemSpecsViewer.exe`.

## Расширение функциональности

- Добавить новую категорию: добавьте датакласс в `models.py`, метод
  `get_x_info()` в `SystemInfoCollector` в `collectors.py`, включите его
  в `collect_all()`, затем добавьте вкладку и сопоставление строк в
  `gui_app.py`.
- Заменить GUI-тулкит: поскольку `collectors.py`/`models.py` не имеют
  зависимостей от GUI, можно построить интерфейс на PyQt/PySide или
  веб-интерфейс поверх того же `SystemInfoCollector`, не трогая слой
  данных.
