# 🖥️ System Specifications Viewer

### Просмотр характеристик компьютера

Современное приложение для мониторинга аппаратного и программного обеспечения компьютера в реальном времени с красивым интерфейсом на **CustomTkinter**: дашборд, детальные страницы по каждому компоненту и встроенный диспетчер задач.

---

## ✨ Возможности

- **📊 Дашборд** — ключевые показатели системы в реальном времени (ЦП, память, диски, сеть, видеокарта, батарея) с кольцевыми индикаторами нагрузки и спарклайнами (графики за 60 секунд)
- **⚡ Процессор** — загрузка ЦП в реальном времени с графиком, загрузка по каждому ядру, частота, количество ядер
- **🧠 Память** — использование ОЗУ и файла подкачки с графиком и прогресс-барами
- **💾 Диски** — использование каждого раздела: занято/всего, прогресс с цветовой индикацией
- **🌐 Сеть** — все адаптеры, IP/MAC-адреса, отправленный/полученный трафик
- **🎮 Видеокарта** — название, драйвер, видеопамять (общая и используемая через NVML), разрешение
- **🖥 Материнская плата** — производитель, модель, версия BIOS, серийный номер
- **🔋 Батарея** — уровень заряда, состояние питания, оставшееся время
- **📋 Диспетчер задач** — полноценная таблица всех процессов: PID, имя, пользователь, ЦП %, память, дисковый ввод-вывод, сетевые подключения, потоки. Сортировка по любому столбцу, поиск/фильтрация, завершение процесса, просмотр свойств (расширенная информация о процессе)
- **🌙 Тёмная и светлая темы** — переключение одним движением, все Canvas-виджеты и Treeview синхронизируются
- **📄 Экспорт отчёта** — JSON (машиночитаемый) или TXT (читаемый) со всей информацией о системе
- **⚡ Оптимизация производительности** — сбор данных в фоновом потоке, обновление только видимой страницы, хеш-проверка перед перерисовкой таблицы процессов, dirty-checking в Sparkline/RingGauge/ProgressStatCard

---

## 🚀 Быстрый старт

```bash
git clone https://github.com/USERNAME/system-specifications-viewer.git
cd system-specifications-viewer
pip install -r requirements.txt
python main.py
```

> **Для Windows** рекомендуется запуск от имени администратора — это даёт доступ к информации о материнской плате/BIOS через WMI и возможность завершать системные процессы.

## 🏗 Архитектура проекта

```
main.py                     # Точка входа, настройка логирования и DPI
  │
  └── gui_app.py            # Главное окно, построение всех страниц, цикл обновлений
        │
        ├── widgets.py      # Переиспользуемые UI-компоненты
        │     ├── Sparkline       # Canvas-график (спарклайн) с dirty-checking
        │     ├── RingGauge       # Кольцевой индикатор с dirty-checking
        │     ├── HeroStatCard    # Крупная карточка-метрика (график + кольцо)
        │     ├── ProgressStatCard # Карточка с прогресс-баром и dirty-checking
        │     ├── FactCard        # Информационная карточка без графика
        │     ├── InfoGrid        # Сетка "параметр → значение"
        │     ├── Sidebar         # Боковая панель навигации
        │     └── PageHeader      # Заголовок страницы
        │
        ├── theme.py        # Цветовая палитра (светлая/тёмная), шрифты, отступы
        ├── icons.py        # Генерация SVG-подобных иконок через Pillow
        │
        └── collectors.py   # Сбор данных о системе
              │
              ├── psutil    # ЦП, память, диски, сеть, процессы, батарея
              ├── wmi       # Материнская плата, BIOS, GPU (Windows only)
              ├── pynvml    # Точная видеопамять NVIDIA (если есть)
              └── platform  # ОС, версия, архитектура

models.py                   # Типизированные датаклассы для всех данных
```

### Оптимизации производительности (v3 — устранение лагов)

1. **Фоновый поток сбора данных** — GUI не блокируется при запросах WMI/psutil
2. **Обновление только видимой страницы** — перерисовываются виджеты только той вкладки, которую видит пользователь
3. **Хеш-проверка в диспетчере задач** — таблица не перестраивается, если набор PID не изменился (решает проблему тормозов при скролле)
4. **Dirty-checking в Canvas-виджетах** — Sparkline и RingGauge не перерисовываются, если данные не изменились
5. **Кэширование** — username процессов (30 с), create_time (навсегда), имя процессора (один раз через WMI), данные материнской платы (один раз при старте), io_counters (раз в 3 с), сетевые подключения (раз в 30 с)

## 📦 Стек технологий

| Технология        | Назначение                              |
| ----------------- | --------------------------------------- |
| Python 3.10+      | Основной язык                           |
| CustomTkinter     | Современный GUI (карточки, темы, скролл)|
| psutil            | Кроссплатформенный мониторинг системы   |
| Pillow            | Генерация иконок навигации              |
| WMI / pywin32     | Материнская плата, BIOS, GPU (Windows)  |
| pynvml            | Точная видеопамять NVIDIA GPU           |
| threading + queue | Фоновый сбор данных (thread-safe)       |

## 📋 Страницы приложения

| Раздел           | Описание                                                     |
| ---------------- | ------------------------------------------------------------ |
| **Дашборд**      | Загрузка ЦП и памяти (кольцо + спарклайн), диски, сеть, GPU, батарея, обзор системы |
| **Система**      | ОС, версия, разрядность, архитектура, время загрузки/работы  |
| **Процессор**    | Общая загрузка + загрузка по ядрам, частота, спецификации    |
| **Память**       | ОЗУ (кольцо, спарклайн, прогресс-бар) и файл подкачки        |
| **Диски**        | Каждый раздел: занято/всего, прогресс-бар с цветом по порогу |
| **Сеть**         | Адаптеры: IP/MAC, статус, отправлено/получено                |
| **Видеокарта**   | Название, драйвер, видеопамять (NVML), разрешение            |
| **Материнская**  | Производитель, модель, версия BIOS, серийный номер           |
| **Батарея**      | Заряд, питание от сети, оставшееся время                     |
| **Диспетчер задач** | Таблица процессов: сортировка, фильтр, завершение, свойства|
